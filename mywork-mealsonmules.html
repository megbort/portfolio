Meals on Mules RESTfulAPI

Video Demo View on GitHub

Meals on Mules is a grocery store company created by my team and myself for a final project in Winter 2022. The company would have both an app and website that sells grocery store products online. Our team consisted of 2 designers, 2 front-end developers and 1 back-end developer to create the API.

Process

I decided to take the role of the back-end developer initially to get more experience. I didn't have much experience in creating a RESTfulAPI but I thought it would be fun to learn.

I first waited to get the mock-ups from our designers, then had a conversation about what was doable for the project and what needed changes. After the final designs were complete I started to plan out my database structure. I initially planned to have some cart functionality but decided to focus mainly on the products API, as this would be the most important part for our project.

As I was creating the database, my team helped me with collecting product data to populate the tables. Once we had the data I started coding out the PHP and SQL to connect the database. After testing the connection with "Postman" I then worked on coding out the required parameters for the project. After these were done I hosted the API on a subdomain and distributed the documentation usage to the front-end developers.

Full API documentation can be found on the Meals on Wheels GitHub Project.
